Cubo

Esse sketch foi feito como atividade da disciplina de matemática aplicada à multimídia 1. A tarefa pedia para que, empregando a instrução box() do processing, demonstrassemos graficamente que o cubo é o paralelepípedo que apresenta o melhor aproveitamento de volume.

Como sabemos, o volume de um paralelepípedo é dado pela seguinte equação: VOLUME = A x B x C

Onde A é o comprimento, B é a largura e C é a altura do objeto, esses parâmetros são inseridos na função box() do processing, porém, ao invés de inserir números nos parâmetros, inseri a posição do mouse no eixo X para o comprimento, a posição do mouse no eixo Y para a altura e para a largura usei o botão direito do mouse para aumentar e o botão esquerdo para diminuir, assim, o usuário consegue modificar as 3 dimensões do paralelepípedo.

Para termos uma ideia de que o cubo é a forma que apresenta o melhor aproveitamento do volume, faremos uma suposição.
Digamos que temos o número 150 para distribuir entre a largura, altura e comprimento de um paraleleípedo, se usarmos o mesmo número para as 3 dimensões do paralelepípedo teremos um cubo de 50 de altura, 50 de largura e 50 de comprimento, isso usa todo o número 150 e conseguimos um volume de 125000, afinal, VOLUME = altura x largura x profundidade. Agora tente disdribuir os 150 com outros valores para as dimensões do objeto e perceba que nenhuma combinação diferente de 50x50x50 conseguirá um resultado maior do que 125000.

Para representar isso visualmente, utilizei o processing em suas funções de manipulação de objetos tridimensionais, o objeto no canvas é manipulável pelo usuário, o manupulador do sketch verá então um paralelepípedo verde quando este for um cubo perfeito e um paralelepípedo vermelho quando este não for um cubo. Aqui eu basicamente já tinha o que precisava, o programa basicamente já cumpria seu papel, apesar de ser quase impossível conseguir o cubo verde sem nenhuma informação adicional além de um cubo verde ou um paralelepípedo vermelho, por um momento pensei em deixar desse jeito e desejar só boa sorte para quem fosse tentar, porém, mudei de ideia e usei a função text() do processing para retornar ao usuário do programa as medidas de altura, largura e comprimento do paralelepípedo, assim ele poderia saber qual dimensão modificar para chegar ao cubo verde e além disso poderia ver que quando o cubo estava desta cor as medidas eram todas iguais, ou seja, um cubo perfeito.
